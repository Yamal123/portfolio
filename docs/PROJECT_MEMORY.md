## 📊 项目画像（长期记忆）

### 项目概述
- **项目名称**: 个人作品网站
- **项目类型**: 全栈个人作品展示平台
- **开发时间**: 2024-2026
- **技术栈**: 
  - 前端: Next.js 15 + React 19 + TypeScript
  - UI: shadcn/ui + Tailwind CSS
  - 后端: Node.js + Fastify
  - 数据库: SQLite
  - 部署: Vercel + GitHub
- **目标用户**: 招聘方、技术同行、个人品牌展示

### 核心功能
1. 作品展示（项目详情、案例介绍）
2. 技能展示（分类展示、能力评估）
3. 后台管理系统（项目管理、技能管理、个人信息管理）
4. 双语支持（中英文切换）
5. 响应式设计（PC/移动端适配）

### 设计理念
- 现代化、专业化的视觉效果
- 字节跳动风格的管理后台
- 简洁清晰的交互体验
- 高性能、SEO 友好

## 🔧 技术债务与优化记录（决策记忆）

### 已解决的问题

#### 类型系统完善（2026-05-18）
- **问题**: Project 接口字段不完整
- **影响**: 17 个 TypeScript 编译错误
- **解决方案**: 
  - 在 `types/admin/index.ts` 添加缺失字段（type_zh, type_en, intro_zh, intro_en, problem_zh, problem_en, action_zh, action_en, result_zh, result_en, keywords, emoji）
  - 统一前台和后台数据结构
- **结果**: TypeScript 编译错误从 30 个降至 0 个

#### 数据导出问题（2026-05-18）
- **问题**: mockProjects 和 mockSkills 未正确导出
- **影响**: 前台页面无法使用模拟数据
- **解决方案**: 
  - 在 `data/projects.ts` 导出 mockProjects
  - 在 `data/skills.ts` 导出 mockSkills
  - 在 `app/page.tsx` 修改 PortfolioSection 导入方式
- **结果**: 编译错误从 19 个降至 2 个

#### 依赖缺失（2026-05-18）
- **问题**: framer-motion 未安装
- **影响**: 动画组件无法使用
- **解决方案**: 安装 framer-motion 依赖
- **结果**: 编译错误降至 0 个

#### 类型安全（2026-05-18）
- **问题**: adapter.ts 和组件中存在隐式 any 类型
- **解决方案**: 
  - 添加 FrontendProject 和 FrontendSkill 类型定义
  - 为所有函数参数添加明确类型
- **结果**: 代码类型安全性提升

#### 代码质量（2026-05-18）
- **问题**: 未使用的导入
- **解决方案**: 
  - 移除 portfolio-section.tsx 中的 Github 和 ScrollArea 导入
  - 创建 ProjectDetailView 组件
- **结果**: 代码更简洁，无冗余

#### API 架构优化（2026-05-18）
- **问题**: API 地址硬编码、重复代码
- **解决方案**:
  - 创建 `lib/api/config.ts` 统一配置
  - 添加 API_TIMEOUT 配置（10秒）
  - 统一 TOKEN_KEY 管理
  - 为所有 API 请求添加超时处理
- **结果**: API 管理更规范，易于维护

#### UI 设计规范（2026-05-18）
- **优化项**:
  - 统一字体颜色层次（gray-900/700/500）
  - 统一组件大小（按钮 h-11, 输入框 h-11）
  - 统一圆角风格（rounded-xl/rounded-2xl）
  - 统一阴影层次（sm/md/lg/xl/2xl）
  - 优化弹窗布局和交互反馈
- **结果**: UI 更专业、一致
