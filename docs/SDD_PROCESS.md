# 软件开发文档 (SDD) - 项目开发流程规范

> 本文档定义了项目的标准化开发流程，确保每次开发、测试都遵循统一的规范

---

## 📋 开发流程总览

```
需求确认 → 方案设计 → 代码开发 → 测试验证 → 文档更新 → 发布上线
    ↓
   每步都需要记录和确认
```

---

## 🎯 开发前准备

### 1. 需求确认清单

在开始任何开发工作前，必须确认以下内容：

- [ ] **需求明确**: 用户需求清晰，无歧义
- [ ] **范围定义**: 明确功能边界和限制
- [ ] **优先级确认**: 确定 P0/P1/P2/P3 优先级
- [ ] **依赖分析**: 识别依赖关系和风险
- [ ] **资源评估**: 评估时间和人力需求

### 2. 环境检查

- [ ] 代码仓库已同步最新版本
- [ ] 开发环境已配置正确
- [ ] 相关依赖已安装
- [ ] TypeScript 编译无错误
- [ ] 测试环境准备就绪

### 3. 文档准备

- [ ] 查看项目记忆文档 (`docs/PROJECT_MEMORY.md`)
- [ ] 查看问题追踪清单 (`docs/ISSUE_TRACKER.md`)
- [ ] 查看相关设计文档
- [ ] 准备开发记录模板

## 📝 方案设计阶段

### SDD-1: 问题分析

在输出任何方案前，必须先分析问题：

```markdown
## 问题分析

### 问题描述
[清晰描述要解决的问题]

### 问题定位
- 文件位置: 
- 影响范围:
- 严重程度: P0/P1/P2/P3

### 根本原因
[分析问题的根本原因]

### 相关问题
- [关联的问题 ID]
- [影响的其他功能]
```

### SDD-2: 方案设计

输出方案时必须包含：

```markdown
## 解决方案

### 方案概述
[简要描述解决方案]

### 技术方案

#### 修改的文件
- [文件路径]

#### 修改内容
```typescript
// 代码示例
```

#### 替代方案
[如果有多个方案，列出其他选项及优缺点]

### 风险评估
- **高风险**: [列出高风险项]
- **中风险**: [列出中风险项]
- **低风险**: [列出低风险项]

### 测试计划
- [ ] 单元测试
- [ ] 集成测试
- [ ] 手动测试项
```

### SDD-3: 方案评审

输出方案后，必须等待确认：

```
⚠️ 方案待确认

请确认以下方案：
1. 问题分析是否准确？
2. 解决方案是否可行？
3. 是否接受该方案？

回复 "确认" 开始开发，或提出修改意见。
```

## 💻 代码开发阶段

### 开发规范

#### 1. 代码编写规范

```typescript
// ✅ 正确示例：添加完整的类型定义
function handleSubmit(data: FormData): Promise<void> {
  // 实现代码
}

// ❌ 错误示例：使用隐式 any
function handleSubmit(data) {
  // 实现代码
}
```

```typescript
// ✅ 正确示例：添加必要的注释（复杂逻辑）
// 使用防抖处理，防止频繁请求
const debouncedSubmit = useDebounce(submitHandler, 300);

// ❌ 错误示例：添加无意义的注释
// 这是一个提交函数
function handleSubmit(data) {
  // 实现代码
}
```

#### 2. Git 提交规范

```bash
# 格式: <类型>: <描述>

# 新功能
git commit -m "feat: 添加用户登录功能"

# Bug 修复
git commit -m "fix: 修复登录失败问题"

# 优化重构
git commit -m "refactor: 重构 API 请求逻辑"

# 文档更新
git commit -m "docs: 更新项目记忆文档"

# 测试相关
git commit -m "test: 添加单元测试"
```

#### 3. 代码检查清单

开发完成后，必须自检：

- [ ] TypeScript 编译无错误
- [ ] 代码符合项目规范
- [ ] 添加了必要的类型定义
- [ ] 异常情况已处理
- [ ] 无未使用的导入
- [ ] 变量命名清晰
- [ ] 函数职责单一

## 🧪 测试验证阶段

### 测试流程

#### 1. 单元测试

```bash
# 运行所有测试
npm test

# 运行特定测试
npm test -- project.test.ts

# 查看测试覆盖率
npm test -- --coverage
```

#### 2. TypeScript 编译检查

```bash
# 编译检查
npx tsc --noEmit

# 查看编译错误详情
npx tsc --noEmit 2>&1 | grep -A 3 "error"
```

## 🔄 流程优化

### 常见问题处理

#### 问题 1: 需求不明确

```
⚠️ 需求不明确

在继续开发前，请确认：
1. 具体要实现什么功能？
2. 预期达到什么效果？
3. 有哪些限制条件？

请提供更详细的需求说明。
```

#### 问题 2: 方案有争议

```
⚠️ 方案待确认

当前方案可能存在以下问题：
- [列出问题]

请确认：
1. 是否接受当前方案？
2. 是否需要修改？
3. 有其他建议？

提供您的选择或修改意见。
```

#### 问题 3: 测试失败

```
⚠️ 测试失败

测试失败信息：
[粘贴错误信息]

请提供：
1. 错误发生的具体场景
2. 预期行为和实际行为
3. 相关代码片段

等待确认后继续排查。
```
