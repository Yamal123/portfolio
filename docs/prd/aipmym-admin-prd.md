## 一、项目概述

### 1.1 项目背景

当前 aipmym.com 个人主页的所有内容均为静态数据硬编码在前端代码中，每次更新内容需要修改代码、重新构建、部署发布，流程繁琐且效率低下。为降低内容运营成本、提升更新效率，需开发独立的后台管理系统，实现数据的在线编辑、实时生效。

### 1.2 项目目标

| 目标 | 指标 | 衡量方式 |
|------|------|---------|
| 内容更新效率提升 | >90% | 从"改代码+部署"变为"表单提交即生效" |
| 运营门槛降低 | 零技术背景可操作 | 无需前端知识即可管理所有内容 |
| 数据安全性 | 权限隔离+操作日志 | 所有操作可追溯 |
| 响应速度 | <200ms | 接口平均响应时间 |

### 1.3 用户角色

| 角色 | 数量 | 权限范围 | 说明 |
|------|------|---------|------|
| 超级管理员 | 1人 | 全部功能权限 | 唯一管理员账号，拥有系统最高权限 |

> **注意**：V1.0 仅支持单一超级管理员角色，不支持多角色、多租户。

## 二、整体架构与核心流程

### 2.1 技术架构

```
┌─────────────────────────────────────────────┐
│              浏览器端 (Admin Frontend)        │
│         React + TypeScript + Ant Design      │
└──────────────────┬──────────────────────────┘
                   │ HTTPS / RESTful API
                   ▼
┌─────────────────────────────────────────────┐
│              服务端 (Admin Backend)           │
│          Node.js + Express / Koa            │
│    JWT认证 │ RBAC权限 │ 操作日志 │ 文件上传   │
├──────────────────┬──────────────────────────┤
│     MySQL        │       Redis              │
│  业务数据存储     │   会话缓存/限流            │
├──────────────────┴──────────────────────────┤
│              OSS / 本地文件系统               │
│            头像/封面图/二维码存储              │
└─────────────────────────────────────────────┘
```

### 2.2 核心业务流程

#### 流程一：登录流程

```
[输入账号密码] → [前端校验] → [请求登录接口] 
→ [后端校验] → [返回JWT Token] → [存储Token到localStorage]
→ [跳转仪表盘]
```

#### 流程二：内容更新流程

```
[进入管理页面] → [点击编辑] → [填写/修改表单] 
→ [前端校验] → [调用保存接口] → [后端写入数据库] 
→ [返回成功] → [Toast提示] → [前台自动刷新显示最新数据]
```

#### 流程三：图片上传流程

```
[选择图片] → [前端预览+压缩] → [调用上传接口] 
→ [后端校验格式/大小] → [存储至OSS] → [返回URL] 
→ [填入表单字段] → [保存时关联]
```

## 三、全局规则

### 3.1 分页规则

| 参数名 | 类型 | 默认值 | 取值范围 | 说明 |
|--------|------|--------|---------|------|
| `page` | number | 1 | >= 1 | 当前页码 |
| `pageSize` | number | 10 | [10, 20, 50, 100] | 每页条数 |

**响应格式**：

```json
{
  "code": 200,
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "pageSize": 10,
    "totalPage": 10
  }
}
```

### 3.2 统一状态码规范

| code | 含义 | HTTP Status | 说明 |
|------|------|------------|------|
| 200 | 成功 | 200 | 请求正常处理 |
| 400 | 参数错误 | 400 | 参数缺失或格式错误 |
| 401 | 未授权 | 401 | 未登录或Token过期 |
| 403 | 禁止访问 | 403 | 无权限操作该资源 |
| 404 | 资源不存在 | 404 | 请求的资源不存在 |
| 429 | 请求过频 | 429 | 触发限流策略 |
| 500 | 服务器错误 | 500 | 内部服务器异常 |

**统一错误响应格式**：

```json
{
  "code": 400,
  "message": "参数校验失败",
  "data": null,
  "errors": [
    { "field": "name", "message": "名称不能为空" }
  ]
}
```

### 3.3 文件上传规则

| 规则项 | 头像 | 封面图 | 二维码 |
|--------|------|-------|-------|
| 格式 | jpg/jpeg/png/webp | jpg/jpeg/png/webp | jpg/png |
| 大小上限 | 2MB | 5MB | 2MB |
| 尺寸建议 | 200x200px | 1920x1080px(16:9) | 300x300px |
| 压缩 | 前端压缩至<1MB | 前端压缩至<2MB | 不压缩 |
| 存储路径 | `/uploads/avatar/` | `/uploads/covers/` | `/uploads/qrcode/` |
| 重命名规则 | `{timestamp}_{random8}.ext` | 同左 | 同左 |

### 3.4 表单校验通用规则

| 校验类型 | 触发时机 | 提示方式 |

## 四、登录模块

### 4.1 登录页面布局

- **页面路径**：`/login`
- **页面标题**：AI PM 后台管理
- **居中卡片式布局**，宽度 420px
- **背景**：深色渐变 + 动态粒子效果（可选）

### 4.2 表单字段定义

| 字段名 | 类型 | 必填 | 占位符 | 校验规则 |
|--------|------|------|--------|---------|
| `username` | text | ✅ | 请输入管理员账号 | 长度 3-20，仅允许字母数字下划线 |
| `password` | password | ✅ | 请输入密码 | 长度 8-32，必须包含大小写字母和数字 |
| `rememberMe` | checkbox | 否 | 记住我 | - |

### 4.3 字段校验详情

**username 校验**：
- 不能为空
- 长度限制：3-20 个字符
- 正则：`^[a-zA-Z0-9_]+$`
- 错误提示："用户名格式不正确"

**password 校验**：
- 不能为空
- 长度限制：8-32 个字符
- 正则：`^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d!@#$%^&*]{8,32}$`
- 错误提示："密码需包含大小写字母和数字，8-32位"

### 4.4 锁定规则

| 条件 | 行为 | 解锁方式 |
|------|------|---------|
| 连续输错密码 5 次 | 账号锁定 30 分钟 | 等待自动解锁或联系开发者 |
| 单 IP 1小时内尝试 20 次 | 该 IP 封禁 1 小时 | 自动解封 |
| Token 过期未刷新 | 强制跳转登录页 | 重新登录 |

锁定期间登录接口返回：

```json
{ "code": 403, "message": "账号已锁定，请30分钟后重试" }
```

### 4.5 记住我逻辑

| 状态 | Token 存储 | 有效期 | 行为 |
|------|-----------|--------|------|
| 勾选 | localStorage | 7 天 | 关闭浏览器后仍保持登录 |
| 未勾选 | sessionStorage | 当前会话 | 关闭浏览器后需重新登录 |

### 4.6 接口说明

**POST /api/auth/login**

请求体：

```json
{
  "username": "admin",
  "password": "Admin@123",
  "rememberMe": true
}
```

成功响应：

```json
{
  "code": 200,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "expiresIn": 604800,
    "user": { "id": 1, "username": "admin", "nickname": "超级管理员" }
  }
}
```

- **页面路径**：`/dashboard`

### 5.2 数据卡片区域（4个）

| 卡片 | 数据来源 | 图标颜色 | 展示字段 |

## 十三、非功能需求

### 13.1 性能需求

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 页面首次加载 | < 2s | 首屏内容渲染完成 |
| 接口响应时间(P50) | < 200ms | 50%的请求在此时间内返回 |
| 接口响应时间(P99) | < 1000ms | 99%的请求在此时间内返回 |
| 并发用户数 | 支持 10 人同时操作 | 单管理员场景足够 |
| 图片加载 | < 1s | CDN 加速后 |

**优化手段**：

- 前端路由懒加载（React.lazy + Suspense）
- 接口请求缓存（SWR / React Query）
- 图片 CDN 加速 + WebP 格式转换
- Gzip / Brotli 压缩
- 数据库查询优化 + 索引建设

### 13.2 安全需求

| 安全项 | 要求 | 实现方式 |
|--------|------|---------|
| 身份认证 | JWT Token + Refresh Token 双令牌 | Access Token 2h，Refresh Token 7d |
| 密码存储 | bcrypt 哈希，cost factor >= 10 | 明文禁止入库 |
| XSS 防护 | 输出转义 + CSP 策略 | 富文本白名单过滤 |
| CSRF 防护 | SameSite Cookie + Token 校验 | API 请求携带 CSRF Token |
| SQL 注入防护 | ORM 参数化查询 | 禁止字符串拼接 SQL |
| 限流 | 接口级别限流 | 登录接口 5次/min，其他 60次/min |
| 操作日志 | 全量记录 | 含操作人、时间、IP、前后数据快照 |
| 敏感操作二次确认 | 删除/恢复等危险操作 | Modal 弹窗确认 |

### 13.3 浏览器兼容性

| 浏览器 | 最低版本 | 备注 |
|--------|---------|------|
| Chrome | >= 90 | 主力支持 |
| Firefox | >= 88 | 支持 |
| Safari | >= 14 | Mac 主力 |
| Edge | >= 90 | Chromium 内核 |
| 移动端 Chrome | >= 90 | Android |
| 移动端 Safari | >= 14 | iOS |

不支持 IE 全系列。
